# プロジェクト作業ルール

## コミットメッセージ
- **日本語**で自然な表現を使う
- 例: 「修正: ボタンがクリックできない問題を解消」
- 例: 「機能追加: メモ入力時にログを分割する機能」

## コメント・説明
- ユーザーへの回答は**日本語**で行う
- コード内のコメントは英語でも日本語でもOK（既存のスタイルに合わせる）

## 作業の進め方
- **複数の指示がある場合、または1つの指示でも**、必ず以下の流れを踏む:
  1. **計画**: 何をするか明確にする
  2. **確認**: ユーザーの承認を得る
  3. **実行**: 承認後に実装
- 重い作業は小分けにして、こまめに確認を取る

## 変更後の確認
- 変更が完了したら、必ず以下を確認する:
  - **チェックポイント**: `git stash` で一時保存（後でロールバック可能）
  - **コミット**: `git commit` で正式に記録
- どちらにするかユーザーに確認してから実行する

## その他
- エラーが発生した場合は原因を調査してから修正する
- ブラウザテストが必要な場合は積極的に実行する
